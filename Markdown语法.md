# 一、基本语法

### 1. 标题（Headings）
要创建标题，请在单词或短语前面添加井号 (`#`)。`#` 的数量代表了标题的级别（1 至 6 级）。

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

#### 可选语法
在文本下方添加任意数量的 `==` 号标识一级标题，或 `--` 号标识二级标题。

```markdown
一级标题
===============

二级标题
---------------
```

> **最佳实践**：请在 `#` 和标题文字之间保留一个空格。

---

### 2. 段落（Paragraphs）
要创建段落，请使用空白行将一行或多行文本进行分隔。

```markdown
这是第一段。

这是第二段（中间有空行）。
```

> **最佳实践**：不要用空格或制表符缩进段落。

---

### 3. 换行（Line Breaks）
在一行的末尾添加**两个或多个空格**，然后按回车键，即可创建一个换行。

```markdown
这是第一行。  
这是第二行。
```

#### 替代方式
使用 HTML 的 `<br>` 标签。

```markdown
这是第一行。<br>
这是第二行。
```

> **最佳实践**：为了兼容性，请在行尾添加“结尾空格”或 HTML 的 `<br>` 标签来实现换行。不推荐使用反斜杠 (`\`) 方式，因为并非所有 Markdown 应用都支持。

---

### 4. 强调（Emphasis）

#### 粗体（Bold）
在单词或短语前后各添加两个星号 (`**`) 或两个下划线 (`__`)。

```markdown
**粗体文本**
__粗体文本__
```

#### 斜体（Italic）
在单词或短语前后添加一个星号 (`*`) 或一个下划线 (`_`)。

```markdown
*斜体文本*
_斜体文本_
```

#### 粗体 + 斜体
在单词或短语前后各添加三个星号 (`***`) 或三个下划线 (`___`)。

```markdown
***粗斜体文本***
___粗斜体文本___
```

> **最佳实践**：在单词或短语中间部分进行强调时，请使用星号而非下划线，以保证兼容性。

---

### 5. 列表（Lists）

#### 有序列表（Ordered Lists）
在每个列表项前添加数字并紧跟一个英文句点 (`.`)。数字不必按数学顺序排列，但列表应当以数字 1 起始。

```markdown
1. 第一项
2. 第二项
3. 第三项
```

> **最佳实践**：请只使用句点 (`.`) 作为分隔符，不要使用右括号 (`)` )。

#### 无序列表（Unordered Lists）
在每个列表项前面添加破折号 (`-`)、星号 (`*`) 或加号 (`+`)。

```markdown
- 第一项
- 第二项
- 第三项
```

> **最佳实践**：不要在同一个列表中混合使用多种分隔符——请选择一种，并始终保持一致。

#### 嵌套列表
缩进一个或多个列表项可创建嵌套列表。

```markdown
1. 第一项
    - 嵌套项 A
    - 嵌套项 B
2. 第二项
```

---

### 6. 链接（Links）
链接文本放在方括号内，链接地址放在后面的圆括号中，链接 title 可选。

```markdown
[链接显示名](<链接地址 "可选标题">)
```

**示例**：
```markdown
[Markdown语法](<https://markdown.com.cn "最好的markdown教程">)
```

#### 网址和 Email 地址
使用尖括号可以把 URL 或 email 地址变成可点击的链接。

```markdown
<https://markdown.com.cn>
<example@email.com>
```

#### 引用类型链接
分为两部分——内联部分和存储在文档其他位置的部分。

```markdown
[链接文本][标签]

[标签]: https://example.com "可选标题"
```

> **最佳实践**：URL 中的空格请使用 `%20` 代替。

---

### 7. 图片（Images）
使用感叹号 (`!`)，然后在方括号内增加替代文本，图片链接放在圆括号里，括号里的链接后可以增加一个可选的图片标题文本。

```markdown
![替代文本](<图片链接 "可选标题">)
```

#### 链接图片
将图片的 Markdown 括在方括号中，然后将链接添加在圆括号中。

```markdown
[![替代文本](图片链接)](跳转链接)
```

---

### 8. 代码（Code）

#### 行内代码
将单词或短语包裹在反引号 (`` ` ``) 中。

```markdown
使用 `printf()` 函数输出。
```

**转义反引号**：如果代码中包含反引号，请使用双反引号 (`` `` ``) 包裹。

```markdown
``Use `code` in your Markdown file.``
```

#### 代码块
将代码块的每一行缩进至少四个空格或一个制表符。

```markdown
    <html>
      <head>
      </head>
    </html>
```
> *注：要创建不用缩进的代码块，请使用围栏式代码块（见下文扩展语法）。*

---

### 9. 引用（Blockquotes）
在段落前添加一个 `>` 符号。

```markdown
> 这是引用内容。
```

**多个段落**：为段落之间的空白行也添加一个 `>` 符号。

```markdown
> 第一段。
>
> 第二段。
```

**嵌套引用**：在要嵌套的段落前添加 `>>` 符号。

```markdown
> 第一级引用。
> > 第二级嵌套引用。
```

**引用中的其他元素**：块引用可以包含其他 Markdown 格式的元素。

```markdown
> #### 标题
> - 列表项
> **粗体**和*斜体*
```

---

### 10. 分隔线（Horizontal Rules）
在单独一行上使用三个或多个星号 (`***`)、破折号 (`---`) 或下划线 (`___`)，并且不能包含其他内容。

```markdown
---
***
___
```

> **最佳实践**：在分隔线的前后均添加空白行。

---

## 二、扩展语法

以下语法在 John Gruber 的原始 Markdown 设计基础上进行了扩展，被许多 Markdown 处理器所支持。

### 1. 表格（Tables）
使用三个或多个连字符 (`---`) 创建每列的标题，并使用管道 (`|`) 分隔每列。

```markdown
| 列1 | 列2 | 列3 |
| --- | --- | --- |
| 内容A | 内容B | 内容C |
| 内容D | 内容E | 内容F |
```

#### 对齐方式
在标题行的连字符左侧、右侧或两侧添加冒号 (`:`) 来控制对齐方式。

```markdown
| 左对齐 | 居中对齐 | 右对齐 |
| :----- | :------: | -----: |
| 文本   |  文本    |   文本 |
```

* **格式化表格中的文字**：可以在表格中添加链接、行内代码和强调，但不能添加标题、块引用、列表、分隔线、图片或 HTML 标签。
* **转义管道字符**：使用 HTML 字符代码 `&#124;` 在表格中显示竖线 (`|`)。

---

### 2. 围栏代码块（Fenced Code Blocks）
在代码块之前和之后的行上使用三个反引号 (`` ``` ``) 或三个波浪号 (`~~~`)。

```markdown
```
{
  "firstName": "John",
  "lastName": "Smith"
}
```
```

#### 语法高亮
在围栏代码块之前的反引号旁边指定一种语言。

```markdown
```json
{
  "firstName": "John",
  "lastName": "Smith"
}
```
```

---

### 3. 脚注（Footnotes）
脚注使您可以添加注释和参考，而不会使文档正文混乱。

* **创建脚注引用**：在方括号内添加插入符号和标识符 (`[^1]`)。
* **定义脚注内容**：使用相同的标识符，后跟冒号和文本 (`[^1]: 脚注内容。`)。

```markdown
这是一个脚注示例[^1]。

[^1]: 这里是脚注的具体内容。
```

> **最佳实践**：脚注可以放在文档中的任何位置（除列表、块引用和表格等元素之外）。

---

### 4. 其他扩展语法
Markdown 官方中文网还提及了以下扩展语法：

* **URL 自动链接**：URL 和 email 地址会被自动转换为可点击的链接。
* **任务列表**：在列表项前添加 `[ ]`（未完成）或 `[x]`（已完成）。
  ```markdown
  - [ ] 计划任务
  - [x] 已完成任务
  ```
* **删除线**：使用两个波浪号 (`~~`) 包裹文字。
  ```markdown
  ~~被删除的文字~~
  ```

---

## 三、最佳实践总结

| 类别      | 建议                           |
| :------ | :--------------------------- |
| **标题**  | `#` 与标题文字之间保留一个空格            |
| **段落**  | 不要用空格或制表符缩进段落                |
| **换行**  | 使用两个空格或 `<br>` 标签            |
| **强调**  | 单词中间（即非首尾部）使用星号而非下划线以防兼容问题   |
| **列表**  | 同一个列表中保持分隔符一致，数字后仅使用英文句点 `.` |
| **链接**  | URL 中的空格请使用 `%20` 代替         |
| **分隔线** | 分隔线的前后均添加空白行                 |
